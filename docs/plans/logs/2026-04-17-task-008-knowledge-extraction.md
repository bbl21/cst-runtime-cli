# task_008 任务记录知识提炼

## 范围

本文档只从 `task_008_ref2_multifreq_realgain_optimization` 的任务记录中提炼知识。

使用的主要记录：

- `tasks/task_008_ref2_multifreq_realgain_optimization/plan.md`
- `runs/run_001/summary.md`
- `runs/run_002/summary.md`
- `runs/run_003/summary.md`
- `runs/run_004/summary.md`
- `runs/run_005/summary.md`
- `runs/run_007/summary.md`
- `runs/run_008/summary.md`
- `runs/run_009/summary.md`

本文档的目的不是复述 run 流水账，而是提炼出：

- 哪些参数方向被验证有效
- 哪些参数方向被明确证伪
- 当前目标函数为什么有效
- 这些 run 记录对后续同类任务有什么可复用价值

## 一、问题画像

### 1. baseline 的真正瓶颈

从 `run_001` 可见，baseline 的主瓶颈不是低频，而是高频侧：

- `13 GHz flatness = 18.423 dB`，是最差频点
- `18 GHz flatness = 14.924 dB`，也是主要风险频点
- `4 GHz` 本身并不是当前主矛盾，它更适合作为“不要严重恶化”的软约束

由此得到的任务画像是：

- 如果继续把四个频点同权看待，很容易被低频或匹配细节牵着走
- 真正该优先压的是 `8 / 13 / 18 GHz`，尤其是 `13 GHz`

### 2. 目标函数重定义是突破前提

任务后半段把主目标改成：

- 最小化 `8 / 13 / 18 GHz` 中最差的 `flatness`
- 次目标是 `8 / 13 / 18 GHz` 平均 `flatness`
- `4 GHz flatness`、`S11`、boresight gain 暂时退成软约束

这次重定义不是展示层调整，而是搜索方向被纠正的关键前提。

从任务记录看，真正的突破 `run_008` / `run_009` 正是在这个新目标函数下出现的。

## 二、参数方向知识

### 1. ridge_bot 是有效方向，但边界很陡

从 `run_002` 和 `run_003` 可以提炼出一条非常清楚的知识：

- `ridge_bot: 11.5 -> 11.0` 是有效方向
- 但继续到 `10.5` 会迅速越界

具体表现：

- `run_002` 已经显示 `8 / 13 / 18 GHz` 都有改善，说明方向是对的
- 但 `run_003` 里 `18 GHz` 匹配直接恶化到 `-3.924 dB`
- 同时 `18 GHz boresight gain` 下降超过 `3 dB`

因此，这条任务记录给出的不是“ridge_bot 越小越好”，而是：

- `ridge_bot` 只能作为有限窗口内的有效调节轴
- 其收益和代价都很陡，不适合大步推进

### 2. ridge_h=28.0 是更健康的稳态改善点

`run_004` 提供了另一类知识：

- `ridge_h: 30.0 -> 28.0` 虽然不是最猛的改善方向
- 但它是“没有明显副作用的健康改善点”

任务记录体现为：

- worst flatness 从 `18.423 dB` 降到 `17.322 dB`
- boresight gain 没有下降
- 匹配也没有失效

这类 run 的价值，不在于它是全局最优，而在于它适合作为后续组合扫描的“安全基底”。

### 3. ridge_bot + ridge_h 组合能再推一步，但会压缩 18 GHz 裕量

`run_005` 说明：

- 把 `run_002` 的有效方向和 `run_004` 的有效方向组合后
- 主目标继续改善到 `17.216 dB`

但任务记录同时给出了重要警告：

- `18 GHz` 的匹配裕量开始被明显压缩
- 该方向虽然还没越界，但已经不适合继续大步同向推进

因此 `run_005` 的知识是：

- ridge 组合可以作为“旧 best”基线
- 但继续沿这条 ridge 主线往前冲，边际收益已经很有限，风险却在上升

### 4. `F < 350` 是被明确证伪的方向

`run_007` 的价值很高，因为它不是“失败 run”，而是明确的方向证伪记录。

任务记录给出的结论很稳定：

- `F=330` 只在 `8 GHz` 有轻微收益
- `13 GHz` 和 `18 GHz` 相比 `run_005` 都回退
- 不应继续沿 `F < 350` 深挖

这条知识的意义在于：

- 后续类似任务里，参数探针不应只看“有没有单频改善”
- 而应看它是否真的改善了当前定义的主目标频带

### 5. `y2` 是本任务中最有效的新参数轴

这是 `run_008` 和 `run_009` 给出的核心知识。

从任务记录看：

- `y2: 60 -> 70` 让 worst target flatness 从 `17.216 dB` 直接压到 `14.509 dB`
- `y2: 70 -> 75` 进一步压到 `14.107 dB`

更关键的是改善结构：

- 最大收益主要来自 `13 GHz`
- `18 GHz` 继续变平
- `8 GHz` 也同步受益

这说明 `y2` 相比 ridge 和 `F`，更直接作用于当前主矛盾所在的高频过渡区。

## 三、机理层推断

### 1. 高频 flatness 的改善来自主瓣拓宽，而不是纯 boresight 抬升

`run_008` 和 `run_009` 的记录都出现了同一个信号：

- `18 GHz` 峰值不再严格位于 boresight
- 峰值偏到了 `theta≈7 deg`

这说明当前 flatness 改善机制不是“把中心点抬高”，而是：

- 通过拓宽高频主瓣
- 减小 `theta <= 15 deg` 邻域内的最大最小差

因此，对这类任务不能只盯 boresight gain。

如果只盯 boresight，很可能会误判一个其实对邻域平坦度更有利的方向。

### 2. `y2` 继续增大时，低频代价会快速显性化

`run_009` 相比 `run_008` 的任务记录显示：

- `8 / 13 / 18 GHz` 还在继续改善
- 但 `4 GHz flatness` 与 `4 GHz boresight gain` 的代价明显放大

因此这里提炼出来的不是“75 一定最佳”，而是：

- `y2` 在 `70 -> 75` 这一区间仍有效
- 但已经开始进入“高频收益继续增加、低频代价快速上升”的区域

这意味着后续搜索不应再做更大的 `y2` 单向粗扫，而应：

- 做局部细化
- 或引入一个专门补偿低频代价的次参数

## 四、搜索策略知识

### 1. 先用单参数粗扫识别活跃方向，再做组合或局部细化

这次任务的记录给出了一条比较清晰的策略链：

1. baseline
2. 单参数粗扫 `ridge_bot`
3. 发现边界，停止继续深推
4. 换扫 `ridge_h`
5. 组合出 `run_005`
6. 用单参数探针证伪 `F`
7. 发现 `y2` 新方向
8. 围绕 `y2` 局部细化

这条链路的价值在于：

- 它不是盲目的网格搜索
- 也不是过早做多参数联调
- 而是“先找有效轴，再组合，再细化”

### 2. 负例 run 的价值不低于 best run

从任务记录看，真正支撑决策的 run 包括：

- `run_003`：告诉你 ridge_bot 不能继续下探
- `run_007`：告诉你 `F < 350` 不值得继续

因此，同类任务不应只沉淀 best run。

负例 run 应该被保留，因为它们定义了停止线和参数边界。

## 五、展示与记录知识

### 1. 展示页不该只展示 best run

这次任务记录的一个明显收获是：

- 如果页面只展示 baseline 和当前 best，用户只能看到“结果”
- 看不到“为什么这样改参”和“哪些方向已经被否掉”

因此更合适的展示结构是：

- 顶层主页面展示多 run 折线
- 方向图按 run 分页聚合
- 同时保留负例 run

### 2. 多 run 过程展示应分成“指标趋势”和“方向图变化”两层

任务记录中实际跑通的展示方法说明：

- flatness 变化最适合用“按频点分线、横轴为 run”的折线图
- 方向图变化更适合直接看 per-run 的 theta-cut 页面

把多个 run、多频点方向图压成热力图，不利于直接比较方向图形态。

### 3. `summary.md` 的价值不只是收尾，而是后续知识提炼入口

这次知识提炼基本可以只靠：

- `plan.md`
- 关键 run 的 `summary.md`

就恢复出完整的决策链。

这说明 `summary.md` 写得好不好，直接决定后续能不能高效做知识回收。

## 六、当前可复用结论

以下结论可直接作为后续同类任务的可复用经验：

- 若 baseline 的主瓶颈集中在某几个频点，应优先重定义主目标，不要机械同权看所有频点
- 先做单参数粗扫识别有效参数轴，再做组合或局部细化
- 对高频邻域 flatness 优化，boresight gain 不应被误当作唯一收益指标
- 负例 run 必须保留，它们定义了参数停止线
- 展示层应同时呈现“多 run 指标趋势”和“多 run 方向图变化”，而不只展示当前 best

## 七、仍然只属于 task_008 的结论

以下内容目前仍应视为任务特定知识，不宜直接升格为全局经验：

- `ref_2` 中 `y2` 是当前最有效参数轴
- `y2=75` 是当前 best-so-far
- `F < 350` 在这个模型上无效
- `18 GHz` 峰值偏到 `theta≈7 deg` 是当前 flatness 改善的表征

这些结论对 `task_008` 很重要，但仍然过于依赖当前模型、当前目标函数和当前角域定义。

## 八、对下一轮任务的直接启发

如果后续继续沿 `task_008` 往下做，任务记录给出的最合理下一步不是：

- 再做更大的 `y2`
- 回到 `F`
- 重新大步扫 ridge

而是：

1. 围绕 `run_009` 做 `y2` 局部细化
2. 引入一个专门补偿 `4 GHz` 低频代价的次参数
3. 保持当前“独立 run + 立即导出 + 立即落盘 + 立即展示”的节奏

这才是从任务记录中真正提炼出的下一步知识。
