# 当前优先清单

> 本清单服务于当前这一周的近期目标。  
> 若与新想法、局部优化或高不确定扩展冲突，以本清单和 `docs/project-goals-and-plan.md` 的当前阶段主攻方向为准。

## 当前唯一主线

当前只集中做一件事：

**把现有生产链收口成一个可迁移、可复用、不依赖上下文和特定 coding agent 的正式交互入口。**

这件事优先于：

- 优化指导功能原型
- 论文研读能力扩展
- 全自动 3D 建模探索
- 大规模 UI 美化或重构
- 与正式入口收口无关的系统泛化工作

## 当前基础目标

### P0-1 定义唯一正式入口

- [ ] 明确唯一正式交互入口是什么
- [ ] 明确它负责哪些阶段，哪些内容不应再由外部脚本或人工兜底
- [ ] 明确正式入口的最小输入、输出和上下文要求

### P0-2 统一主生产链

- [ ] 盘点当前 MCP、Skill、知识系统、计划系统的实际使用点
- [ ] 将 run 创建、执行、导出、对比、状态落盘统一到同一主链路
- [ ] 确认不同入口不会再走出不同生产结果

### P0-3 清理临时脚本与旁路流程依赖

- [ ] 找出当前仍承担生产职责的 `tmp/`、一次性脚本或旧入口
- [ ] 将可保留内容迁回正式链路，不能保留的标记为非生产路径
- [ ] 确认正式流程不依赖特定 coding agent 的隐式记忆

### P0-4 打通四类系统协同

- [ ] MCP 负责稳定接口与执行能力
- [ ] Skill 负责正式方法和流程约束
- [ ] 知识系统负责规则、经验、长期共识的分层落点
- [ ] 计划系统负责当前主线、阶段目标和清单维护

### P0-5 验证可迁移性

- [ ] 在低上下文条件下跑通一次完整流程
- [ ] 验证标准 `run_xxx/{projects,exports,logs,stages,analysis}` 产物完整落盘
- [ ] 验证错误分支、状态文件和知识回收路径都可用

## 进阶目标

### P1-1 优化指导功能原型

进入条件：

- [ ] P0 全部完成
- [ ] 正式生产链已不依赖临时脚本、旁路流程、特定 agent 或隐式上下文

原型目标：

- [ ] 能基于建模代码识别当前天线对象、结构线索和已有参数信息
- [ ] 能主动向用户询问缺失的天线需求、约束和目标指标
- [ ] 能围绕具体需求寻找并研读相关论文
- [ ] 能输出面向当前任务的结构化优化指导，而不是泛泛综述

## 当前明确不做

- [ ] 不把优化指导功能和基础目标并行推进
- [ ] 不插入新的建模主线
- [ ] 不扩展到其他器件类型
- [ ] 不重做大规模 UI
- [ ] 不顺手重构与正式入口无关的模块
- [ ] 不为了“看起来先进”而加入未收口的新能力

## 使用规则

- 每次开始新任务前，先判断它是否直接支撑 `P0`
- 若不能直接支撑 `P0`，默认归类为“后续计划”或“观察项”
- 只有在 `P0` 完成后，才允许进入 `P1`
- 若本清单失效，应同步更新本文件和 `docs/project-goals-and-plan.md`
