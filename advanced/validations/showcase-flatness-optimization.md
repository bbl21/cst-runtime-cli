# 近轴方向图平坦度优化展示案例

本案例展示 CST_MCP 如何用“LLM + Skill + CLI/runtime + 标准 run 记录”的方式推进一个真实 CST 参数化优化任务。案例来自 `task_008_ref2_multifreq_realgain_optimization` 的知识提炼，仅保留脱敏后的指标、参数方向和决策链，不包含 CST 原始工程、run 产物或本地路径。

## 任务目标

优化对象是一个已有参数化 CST 工程。任务目标不是重新建模，而是在既有参数基础上改善多频点近轴方向图平坦度。

优化关注区间：

- 频点：`8 / 13 / 18 GHz`
- 角域：近轴邻域，`theta <= 15 deg`
- 主指标：目标频点中最差的 flatness
- 软约束：低频 `4 GHz`、S11 匹配、boresight gain 不应严重恶化

## 基线问题

baseline 显示主要瓶颈集中在高频侧：

| 频点 | baseline flatness |
| --- | ---: |
| 13 GHz | 18.423 dB |
| 18 GHz | 14.924 dB |

因此任务没有继续把所有频点机械同权处理，而是把主目标重定义为：优先压低 `8 / 13 / 18 GHz` 中的 worst flatness，再看平均 flatness 和软约束。

## 优化过程

优化采用“先找有效参数轴，再组合，再局部细化”的策略。

| 阶段 | 结论 |
| --- | --- |
| baseline | 确认主瓶颈在 13 GHz 和 18 GHz |
| `ridge_bot` 探针 | `11.5 -> 11.0` 有效，但继续到 `10.5` 会明显越界 |
| `ridge_h` 探针 | `30.0 -> 28.0` 是较健康的稳态改善点 |
| ridge 组合 | worst target flatness 改善到 `17.216 dB`，但 18 GHz 匹配裕量收窄 |
| `F < 350` 探针 | 被证伪，只对单频有轻微收益，不能改善主目标 |
| `y2` 探针 | 成为最有效新参数轴，直接改善高频过渡区 |
| `y2` 局部细化 | `70 -> 75` 继续改善目标频点，但低频代价开始显性化 |

## 关键结果

| 节点 | worst target flatness |
| --- | ---: |
| baseline | 18.423 dB |
| ridge 健康改善点 | 17.322 dB |
| ridge 组合基线 | 17.216 dB |
| `y2 = 70` | 14.509 dB |
| `y2 = 75` | 14.107 dB |

最终 best-so-far 将目标频点 worst flatness 从 `18.423 dB` 压到 `14.107 dB`，改善约 `4.316 dB`。

## 方法价值

这个案例体现了 CST_MCP 的几个核心能力：

- 用标准 run 结构保存每次尝试的输入、输出、状态和摘要。
- 用 Skill 约束 agent 不直接修改参考工程，避免污染蓝本。
- 用 runtime/CLI 工具沉淀参数读写、结果读取、远场处理和指标计算。
- 用负例 run 记录停止线，避免重复探索已经证伪的方向。
- 用知识提炼文档把一次优化任务转化为后续可复用经验。

## 可复用经验

- 如果 baseline 的主瓶颈集中在少数频点，应先重定义主目标，而不是机械同权优化全部频点。
- 高频 flatness 的改善不一定来自 boresight gain 抬升，也可能来自主瓣拓宽。
- 负例 run 和 best run 同样重要，前者定义参数边界和停止线。
- 展示时应同时呈现多 run 指标趋势和方向图变化，而不是只展示最终 best。
